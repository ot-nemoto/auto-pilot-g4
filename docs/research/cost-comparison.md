# ランニングコスト比較

## 前提

- Gemma4 4B モデル使用
- 1テスト実行 = コールドスタート込みで約15分と仮定
- 月100回実行想定

---

## 構成別の単価比較

### CPU推論（Fargate vs Batch on EC2）

| 構成 | スペック | 時間単価 | 15分コスト |
|------|----------|----------|-----------|
| ECS Fargate（オンデマンド） | 8vCPU / 16GB | $0.395/h | **$0.099** |
| ECS Fargate Spot | 8vCPU / 16GB | $0.127/h | **$0.032** |
| AWS Batch + EC2 c5.2xlarge（オンデマンド） | 8vCPU / 16GB | $0.340/h | **$0.085** |
| AWS Batch + EC2 c5.2xlarge（Spot） | 8vCPU / 16GB | ~$0.102/h | **$0.026** |

→ CPU推論ならFargateとBatchで**ほぼ同じ**。Spot同士でも誤差範囲。

---

### GPU推論（Batchのみ）※Fargateは不可

| 構成 | スペック | 時間単価 | 15分コスト |
|------|----------|----------|-----------|
| AWS Batch + g4dn.xlarge（オンデマンド） | 4vCPU / 16GB / T4 GPU | $0.526/h | **$0.132** |
| AWS Batch + g4dn.xlarge（Spot） | 4vCPU / 16GB / T4 GPU | ~$0.190/h | **$0.048** |
| AWS Batch + g5.xlarge（オンデマンド） | 4vCPU / 16GB / A10G GPU | $1.006/h | **$0.252** |

→ GPUはCPUより高いが、**推論が10〜50倍速**になるためテスト時間自体が短縮される。

---

## 月100回実行のトータルコスト

| 構成 | 15分×100回 | 備考 |
|------|------------|------|
| Fargate オンデマンド | **$9.9/月** | GPU不可 |
| Fargate Spot | **$3.2/月** | GPU不可 |
| Batch + c5.2xlarge Spot | **$2.6/月** | GPU不可 |
| Batch + g4dn.xlarge オンデマンド | **$13.2/月** | GPU推論（高速） |
| Batch + g4dn.xlarge Spot | **$4.8/月** | GPU推論（高速）+ 中断リスクあり |

※ EC2の最低課金は1分。その後は秒単位。

---

## 結論

```
CPU推論で良い場合:
  Fargate Spot ≒ Batch + EC2 Spot（ほぼ同じ）
  → どちらでも月数ドル程度

GPU推論が必要な場合:
  Batch + g4dn Spot が唯一の現実的な選択肢
  → 月$5程度（Spot）〜$13程度（オンデマンド）
```

**AWS Batch自体のサービス料金はゼロ**。EC2代のみ。

## 付帯コスト（共通・少額）

| リソース | 概算 |
|----------|------|
| API Gateway | $3.50/100万リクエスト → 100回なら $0.00035 |
| Lambda | 無料枠内（ほぼ無料） |

## GPU推論が本当に必要か

Gemma4 4B の CPU 推論速度（参考値）:

| ハードウェア | トークン生成速度 | テストコード生成（500トークン）|
|-------------|----------------|-------------------------------|
| 8 vCPU（c5.2xlarge） | 5〜15 tok/sec | **33〜100秒** |
| T4 GPU（g4dn.xlarge） | 50〜100 tok/sec | **5〜10秒** |
| A10G GPU（g5.xlarge） | 100〜200 tok/sec | **2〜5秒** |

テスト自体（Playwright実行）が30秒〜数分かかるため、
**CPU推論でも許容範囲内**な可能性が高い。
まずはCPU（Fargate or Batch）で始めて、遅ければGPUに移行する判断でOK。
